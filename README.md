```bash
🧩 구현 요약

1. 라벨 정규화 방식

입력 데이터에서는 다양한 형태의 라벨이 사용되므로, 프로그램 내부에서는 일관된 기준을 유지하기 위해 라벨 정규화를 수행하였다.

* `'+'`, `'cross'` → `Cross`
* `'x'` → `X`

이를 통해 데이터의 표현 방식 차이로 인해 발생할 수 있는 비교 오류를 방지하였다.

---

2. MAC 연산 구현

MAC(Multiply-Accumulate) 연산은 입력 패턴과 필터의 각 원소를 위치별로 곱한 뒤, 그 결과를 모두 더하는 방식으로 구현하였다.

total += matrix_a[i][j] * matrix_b[i][j]

이 연산은 이중 반복문을 사용하여 직접 구현하였으며, 외부 라이브러리 없이 순수 Python으로 처리하였다.

MAC 연산 결과는 두 패턴 간의 유사도를 나타내며, 값이 클수록 두 패턴이 더 유사함을 의미한다.

---

3. 판정 방식 및 동점 처리 정책

각 패턴에 대해 Cross 필터와 X 필터의 MAC 점수를 각각 계산한 후, 두 값을 비교하여 최종 판정을 수행하였다.

* Cross 점수 > X 점수 → `Cross`
* Cross 점수 < X 점수 → `X`
* 두 점수 차이가 매우 작은 경우 → `UNDECIDED`

부동소수점 오차를 고려하여 다음과 같은 epsilon 기반 비교를 적용하였다.

if abs(cross_score - x_score) < 1e-9:
    result = "UNDECIDED"

이를 통해 미세한 계산 오차로 인한 잘못된 판정을 방지하였다.

---

📊 결과 리포트

1. FAIL 케이스 분석

본 실험에서는 모든 테스트 케이스가 정상적으로 PASS 되어 FAIL 케이스는 발생하지 않았다.

이는 다음과 같은 설계 요소가 올바르게 적용되었기 때문이다.

* 라벨 정규화를 통해 입력 데이터의 불일치를 제거
* Cross / X 필터를 동시에 비교하는 구조로 정확한 판정 수행
* epsilon 기반 비교를 통해 부동소수점 오차를 안정적으로 처리

특히, epsilon 비교를 적용하지 않을 경우 미세한 수치 차이로 인해 동일한 값임에도 다른 결과로 판정될 수 있으나, 이를 방지하여 모든 테스트에서 정확한 결과를 얻을 수 있었다.

---

2. 성능 분석 및 시간 복잡도

실행 결과, 패턴 크기 증가에 따른 연산 시간은 다음과 같은 경향을 보였다.

* 5×5 → 25 연산
* 13×13 → 169 연산
* 25×25 → 625 연산

이는 MAC 연산이 이중 반복문 구조로 이루어져 있기 때문이다.

for i in range(N):
    for j in range(N):

따라서 전체 연산 횟수는 N × N, 즉 N²에 비례하게 된다.

실제 측정된 시간 또한 다음과 같은 특징을 보였다.

* 입력 크기가 증가할수록 연산 시간이 증가
* 연산 횟수 증가 비율과 유사한 시간 증가 패턴 확인

이를 통해 MAC 연산의 시간 복잡도가 다음과 같음을 확인할 수 있다.

O(N²)

즉, 입력 크기가 커질수록 연산량이 제곱 형태로 증가하며, 이는 대규모 AI 연산에서 NPU와 같은 병렬 처리 장치가 필요한 이유를 설명해준다.
```
