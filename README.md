2.1 MAC(Multiply-Accumulate) 연산 원리

Python
for i in range(m1.size):
    for j in range(m1.size):
        total += m1.get_value(i, j) * m2.get_value(i, j)
MAC 연산은 두 행렬의 동일 위치 원소를 곱해 누적하는 과정입니다. 이는 두 패턴 간의 공간적 일치성을 측정하는 것으로, 곱해지는 두 값이 모두 클수록 누적 합이 커집니다. 따라서 점수가 높을수록 필터의 특징과 패턴이 기하학적으로 더 유사하다고 판단합니다.

2.2 부동소수점 오차(Epsilon) 발생 원인

컴퓨터는 모든 숫자를 0과 1로만 이해합니다. 하지만 우리가 쓰는 0.1 같은 숫자는 0과 1만 써서 표현하려고 하면 딱 떨어지지 않고 0.10000000000000001... 처럼 끝없이 이어지는 숫자가 되어버립니다. 이를 방지하기 위해 아주 미세한 차이는 무시하고 **동일한 값으로 인정해 주는 안전장치(에실론)**를 사용하여 정확한 판정을 내리도록 했습니다.

측정 구간: measure_time 함수 내에서 mac_operation이 수행되는 순수 연산 구간만 측정합니다. 파일 I/O 및 출력 시간을 배제하여 알고리즘 순수 성능을 도출했습니다 (기준 13)

분석: 데이터 크기가 5에서 25로 5배 증가할 때, 연산 횟수는 25에서 625로 25배(5^2) 증가합니다. 이는 이중 반복문 구조로 인해 **시간 복잡도가 O(N^2)**임을 수치적으로 증명합니다.

4.2 1000x1000 대규모 연산 시의 문제

메모리 병목: 100만 개의 요소를 가진 리스트는 파이썬 객체 구조상 메모리에 산재되어 **캐시 미스(Cache Miss)**를 유발합니다.

해결책: NumPy를 사용하면 데이터를 메모리에 빈틈없이 옆으로 딱 붙여서 저장할 수 있습니다. 이렇게 하면 컴퓨터가 숫자를 찾으러 돌아다니는 시간을 줄여주고, 한 번의 동작으로 여러 개의 숫자를 동시에 계산하는 '도장 찍기' 방식이 가능해져서 100만 개의 연산도 순식간에 처리할 수 있습니다. 넘파이 같은 도구가 없다면, 파이썬의 array 기능을 써서 2차원 표 형태의 데이터를 긴 한 줄로 쭉 늘어뜨립니다. 이렇게 데이터를 한 줄로 딱 붙여서 저장하면, 컴퓨터가 다음 숫자를 찾으러 돌아다닐 필요 없이 순서대로 빠르게 읽을 수 있어 100만 개가 넘는 큰 연산도 훨씬 수월하게 처리할 수 있습

4.3 CPU vs NPU 병목 현상

1. CPU 병목: 제어 로직의 과부하

원인: CPU는 한 번에 하나의 명령어를 해석하고 실행하는 순차 구조입니다.

현상: 실제 계산 시간보다 다음 명령어를 불러오고, 조건문을 판단하고, 실행 순서를 정하는 등의 **준비 과정(제어 오버헤드)**에 더 많은 시간이 소요되어 성능이 제한됩니다.

2. NPU 병목: 데이터 공급 속도의 한계

원인: 수천 개의 연산기(ALU)가 동시에 작동하여 계산 속도가 극도로 빠릅니다.

현상: 연산 장치의 처리 속도에 비해, 메모리로부터 데이터를 읽어오는 **전송 속도(메모리 대역폭)**가 이를 뒷받침하지 못해 연산 장치가 데이터를 기다리며 멈추는 현상이 발생합니다.
